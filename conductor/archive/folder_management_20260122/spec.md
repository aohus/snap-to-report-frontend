# Specification - 작업 및 현장별 폴더 관리 시스템 구축

## 1. 개요
사용자가 생성한 수많은 작업(Job)들을 현장 단위나 카테고리별로 폴더를 만들어 체계적으로 관리할 수 있는 기능을 제공합니다.

## 2. 목표
- 사용자가 직접 폴더를 생성하고 이름을 수정할 수 있음.
- 기존 작업들을 특정 폴더로 이동시키거나 폴더에서 뺄 수 있음.
- 폴더 내에서 해당 작업들만 필터링하여 볼 수 있는 내비게이션 제공.

## 3. 상세 기능
- **폴더 CRUD:** 폴더 생성, 이름 변경, 삭제 기능.
- **작업 이동:** 작업을 폴더로 드래그하여 이동하거나 메뉴를 통해 이동.
- **폴더 뷰:** 특정 폴더를 선택했을 때 해당 폴더에 속한 작업들만 리스트에 노출.

## 4. 기술적 고려 사항
- **API:** 백그라운드 서버에 폴더 관련 API가 구축되어 있는지 확인 필요 (없을 경우 로컬 메타데이터나 특정 필드 활용).
- **State:** Zustand를 사용하여 현재 선택된 폴더 및 폴더 구조를 전역 관리.
